# Release Assets

Repository files appear after PR merge. GitHub Release downloadable assets are separate uploads attached to a tag.

Suggested release tag:

```text
v1.0.0-rc1
```

Suggested release title:

```text
TopoAccess v1.0.0-rc1
```

Suggested assets:

- `release/topoaccess_prod_v39/release_manifest.json`
- `release/topoaccess_prod_v39/release_notes.md` if present
- Source or release archive generated by the release workflow
- `checksums.txt`

Create the release after the PR is merged and the tag exists:

```bash
gh release create v1.0.0-rc1 \
  --repo mikeanderson42/TopoAccess \
  --title "TopoAccess v1.0.0-rc1" \
  --notes-file release/topoaccess_prod_v39_1/release_notes.md
```

Upload assets:

```bash
gh release upload v1.0.0-rc1 \
  release/topoaccess_prod_v39/release_manifest.json \
  checksums.txt \
  --repo mikeanderson42/TopoAccess
```

Do not upload private caches, model files, GGUF files, `.env` files, logs, or local-only runtime outputs.
