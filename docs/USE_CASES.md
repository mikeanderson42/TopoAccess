# Use Cases

## Best Fit

- Large or medium repositories where agents repeatedly ask for repo facts.
- Repos with meaningful tests, scripts, docs, release commands, or artifact metadata.
- Agent workflows that need compact Codex briefs, impacted tests, command lookup, post-edit validation, or provenance-backed answers.
- Local-first teams that want deterministic repo tools before optional model-backed synthesis.

## Not A Good Fit

- Tiny repos where reading the whole project is cheaper than maintaining an index.
- Workflows that need a fully hosted cloud agent product.
- Teams that expect universal production savings without measuring their own task mix.
- Tasks that require code execution, network calls, or private services that are unavailable locally.

## Who Benefits

- Codex users who want compact briefs before edits.
- Claude Code, Cursor, Aider, OpenClaw/OpenHands, and generic agent users who want a model-free repo sidecar.
- HTTP and stdio tool users who need schemas and deterministic exact lookup.
- Maintainers who want release, audit, and post-edit validation guardrails.
