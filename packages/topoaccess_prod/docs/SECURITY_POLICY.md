# Security Policy

TopoAccess is read-first and plan-first.

- No hidden writes.
- Dry-run installers by default.
- No external network required for local use.
- Exact lookup bypasses the model.
- Preferred model remains category-gated.
- Provenance is required.
- Unsupported queries abstain.
- Nonpreferred model use fails release gates.
- Secrets must not be included in prompt packs, release artifacts, or sync commits.
